Multiple Time Series

What are the long term trends in complaint generation? How do complaints vary over the course of the week? By rolling up total daily complaints by each complaint type and plotting those series using dygraphs.js, it is easy to spot emerging trends as well as determine how to staff for complaint volumes over the course of a week.

  • Data: CSV with multiple time series

Data was processed using R and visualized using dygraphs.js.


##Specify packages
  library(sqldf)
  library(dygraphs)
  library(xts)
  op <- options(gvis.plot.tag="chart")

##Load in data

  setwd("/Users/jeffreychen/Google Drive/PIF - Other/013_CFPB_Workshop/raw_data")
  data <- read.csv("Consumer_Complaints.csv")
  colnames(data)<-gsub("\\.","",colnames(data))

Now, the data will be rolled up by date and product to create a time series.

#Process Issue to bank data
    ts <- sqldf("SELECT Datereceived, Product as product, count(Datereceived) count
                   FROM data
                   GROUP BY Datereceived,product")

The data is in long form. In order to put it into a panel time series, the data needs to be transformed to wide form.

    w <- reshape(ts, 
                 timevar = "product",
                 idvar = c("Datereceived"),
                 direction = "wide")
    
    colnames(w)<-gsub("count.","",colnames(w))
    colnames(w)<-gsub(" ","_",colnames(w))  
  
  w[is.na(w)]<-0
  w$Datereceived <- as.Date(w$Datereceived,"%m/%d/%Y")
  w<-w[order(w$Datereceived),]

  ##Put data into extensible time series format in order to use the Dygraphs library
  graph <- as.xts(w,order.by = w$Datereceived)

Calendar Heatmap

Seasonality patterns are often tied to days of the week or time of the year. When are the worst hit times? When can we anticipate a rush? Are we plateauing? Calendar heatmaps are a simple way of identifying critical points of time when activity changes. Using SQL code, the raw data is rolled up by date. The GoogleVis library that taps into the Google Charts API to create a calendar heatmap.

  • Data: CSV of time series 
  
  ##Specify packages
      library(sqldf)
      library(googleVis)

  ##SQL data processing
      calendar_data<-sqldf("SELECT Datereceived as date, count(Datereceived) count
                   FROM data
                       GROUP BY date")
      calendar_data$date <- as.Date(calendar_data$date,"%m/%d/%Y")
      calendar_data <- calendar_data[ order(calendar_data$date),]

  ##Create Google Charts Object
      Cal <- gvisCalendar(calendar_data, 
                        datevar="date", 
                        numvar="count",
                        options=list(
                          title="Daily complaints to CFPB",
               
                          calendar="{yearLabel: { fontName: 'Arial',
                                   fontSize: 32, color: 'grey', bold: true},
                                   cellSize: 10,
                                   cellColor: { stroke: 'grey', strokeOpacity: 0.2 },
                                   focusedCellColor: {stroke:'grey'}}")
    )
  ##Initialize Charts Object
    plot(Cal)

Bubbles

Which types of financial products and resolution strategies result in the lowest complaint rates? Are there dramatic differences across groups? Using bubble charts, these rates can be visualized through floating proportional symbols. The human eye can easily identify trends by color and area.
